Why prompt management?
the problem with hardcoded prompts

Look at your Day 1 capstone — the QA Architect system prompt is hardcoded in 11_capstone.py. Every time a QA lead wants to tweak the persona, tone, or instructions, they need to:

🔴
Without prompt management
Edit Python file → git commit → code review → merge → CI/CD pipeline → deploy → wait. A 5-word change to a prompt takes hours or days and requires engineering involvement.
🟢
With Langfuse prompt management
Edit prompt in Langfuse UI → click Save → click "Set as production" → done in 30 seconds. Your running app picks it up on the next request. Zero engineering involvement needed.

You also get version history (every change is tracked), rollback (one click to revert), A/B testing (different labels per environment), and prompt-to-trace linking (see exactly which version of a prompt generated which output).

Real impact: Langfuse traces every generation with its prompt_name and prompt_version. When something goes wrong in production, you can see exactly which prompt version caused it — not just "the prompt was wrong".

Core concepts: versions & labels
the mental model before touching code

Langfuse prompts have two orthogonal concepts you need to understand before writing any code:

Versions vs Labels — how they work together
v1
Initial version
old prod
→
v2
Added examples
staging
→
v3
Latest change
production latest
• Versions — immutable, numbered automatically (1, 2, 3…). Every save creates a new version. You can never edit or delete a version.
• Labels — mutable pointers to versions. production = what your app pulls by default. latest = auto-maintained by Langfuse. You create custom ones freely.
• Your code references labels, never version numbers. Moving production from v2 to v3 requires zero code change.
Prompt TypeFormatBest for
Text prompt A single string. Variables use {{variable}} syntax. System prompts, simple completions, one-shot instructions
Chat prompt Array of [{role, content}] messages with variables Multi-turn chat templates, few-shot examples baked in

Create prompts via the Python SDK
01_create_prompts.py — push text & chat prompts

You can create prompts programmatically — useful for seeding a project or automating prompt pipelines. Create 01_create_prompts.py inside a new day2/ folder:

BASH setup 
$ mkdir day2 && cd day2
PYTHON day2/01_create_prompts.py 
from dotenv import load_dotenv
load_dotenv("../.env")   # .env is one level up in langfuse-tutorial/

from langfuse import get_client

langfuse = get_client()

# ─────────────────────────────────────────────────────────────────
# PROMPT 1: Text prompt (single string, good for system messages)
# Variables use {{double_curly_brace}} syntax
# ─────────────────────────────────────────────────────────────────
text_prompt = langfuse.create_prompt(
    name="qa-architect-system",
    type="text",
    prompt=(
        "You are an experienced QA Architect with {{experience_years}} years of "
        "software industry experience. You specialise in {{specialisation}}. "
        "Your communication style is {{style}}. "
        "Always provide practical, actionable advice with concrete examples."
    ),
    config={                          # model parameters stored alongside prompt
        "model":       "llama3.2",
        "temperature": 0.5,
        "max_tokens":  500,
    },
    labels=["production"],           # immediately mark as production
)
print(f"✓ Text prompt created:  {text_prompt.name} v{text_prompt.version}")


# ─────────────────────────────────────────────────────────────────
# PROMPT 2: Chat prompt (array of messages — great for few-shot)
# ─────────────────────────────────────────────────────────────────
chat_prompt = langfuse.create_prompt(
    name="qa-test-case-chat",
    type="chat",
    prompt=[
        {
            "role": "system",
            "content": (
                "You are a QA Architect specialising in test case design. "
                "Generate test cases for the {{feature_type}} feature described. "
                "Use {{test_format}} format. Cover: happy path, edge cases, and error scenarios."
            ),
        },
        {
            "role": "user",
            "content": "Feature: {{feature_description}}",
        },
    ],
    config={
        "model":       "llama3.2",
        "temperature": 0.3,   # lower temp = more consistent test cases
    },
    labels=["production"],
)
print(f"✓ Chat prompt created:  {chat_prompt.name} v{chat_prompt.version}")
print("\nCheck Langfuse UI → Prompt Management to see both prompts.")
BASH run it
$ python 01_create_prompts.py
In the dashboard: Go to Prompt Management in the left sidebar. You'll see both prompts — qa-architect-system and qa-test-case-chat — each at version 1 with the production label. Click into either one to see the template, variables, config, and version history.
create_prompt() is idempotent only by name: Calling it again with the same name creates a new version, it does not update in place. This is intentional — every call creates an immutable version. If you run 01_create_prompts.py twice, you'll have v1 and v2 of each prompt.

Create & edit prompts in the UI
the no-code workflow for non-engineers

The UI is where non-technical team members — product managers, QA leads, domain experts — iterate on prompts without ever touching code. Here's the exact workflow:

1
Open Prompt Management
In your Langfuse project sidebar → Prompt Management. Click + New Prompt (or click an existing one to edit).
2
Edit the prompt text
The editor supports {{variable}} syntax. Variables are automatically detected and shown in the Variables panel on the right. Try editing qa-architect-system — change {{experience_years}} to a fixed number like "20".
3
Test in the Playground
Click Playground tab. Fill in variable values and run it against a model. This tests your prompt before deploying — no code needed.
4
Save as new version
Click Save. This creates a new immutable version (e.g. v2). It automatically gets the latest label. The production label stays on v1 until you move it.
5
Promote to production
In the version list, click v2 → Add Label → type production → confirm. Your running app will pick this up on the next fetch (within cache TTL, default 60s). Zero deployment needed.
6
Rollback if needed
Move the production label back to v1. Done in one click. The bad v2 is still in history — you can always inspect it.

get_prompt() and compile()
02_use_prompts.py — pull from Langfuse at runtime

This is the core runtime pattern. get_prompt() fetches the prompt from Langfuse (cached for 60s by default). compile() fills in the {{variables}}. The result is a string or message list ready to send to your LLM.

qa-architect-system
production v1
You are an experienced QA Architect with {{experience_years}} years of
software industry experience. You specialise in {{specialisation}}.
Your communication style is {{style}}.
Always provide practical, actionable advice with concrete examples.
{{experience_years}} {{specialisation}} {{style}}

Create day2/02_use_prompts.py:

PYTHON day2/02_use_prompts.py 
from dotenv import load_dotenv
import os
load_dotenv("../.env")

from langfuse import get_client, observe, propagate_attributes
from openai import OpenAI

langfuse = get_client()
client   = OpenAI(base_url=os.getenv("OLLAMA_BASE_URL"), api_key="ollama")

# ─────────────────────────────────────────────────────────────────
# PATTERN 1: Text prompt — get, compile, use
# ─────────────────────────────────────────────────────────────────

# get_prompt() fetches from Langfuse (cached 60s)
# No label = uses 'production' label automatically
system_prompt_obj = langfuse.get_prompt("qa-architect-system")

print(f"Loaded: {system_prompt_obj.name} v{system_prompt_obj.version}")
print(f"Labels: {system_prompt_obj.labels}\n")

# compile() fills {{variables}} with actual values → returns a string
compiled_system = system_prompt_obj.compile(
    experience_years="20",
    specialisation="test automation and CI/CD pipelines",
    style="concise and practical",
)
print("Compiled system prompt:")
print(compiled_system)
print()


# ─────────────────────────────────────────────────────────────────
# PATTERN 2: Chat prompt — get, compile, use as messages list
# ─────────────────────────────────────────────────────────────────
chat_prompt_obj = langfuse.get_prompt("qa-test-case-chat")

# For chat prompts, compile() returns a list of {role, content} dicts
compiled_messages = chat_prompt_obj.compile(
    feature_type="login",
    test_format="Gherkin (Given/When/Then)",
    feature_description="User login with email and password. Max 3 attempts before lockout.",
)
print("Compiled chat messages:")
for msg in compiled_messages:
    print(f"  [{msg['role']}]: {msg['content'][:80]}...")
print()


# ─────────────────────────────────────────────────────────────────
# PATTERN 3: Get specific version or label
# ─────────────────────────────────────────────────────────────────
# Get a specific version number
v1_prompt = langfuse.get_prompt("qa-architect-system", version=1)
print(f"Specific version: v{v1_prompt.version}")

# Get a specific label (e.g. staging)
# latest_prompt = langfuse.get_prompt("qa-architect-system", label="latest")

# Read model config stored with the prompt
print(f"Config from prompt: {system_prompt_obj.config}")
model      = system_prompt_obj.config.get("model", os.getenv("OLLAMA_MODEL"))
temperature = system_prompt_obj.config.get("temperature", 0.5)

print(f"\nUsing model={model}, temperature={temperature}\n")

# ─────────────────────────────────────────────────────────────────
# PATTERN 4: Use the compiled prompt in an actual LLM call
# ─────────────────────────────────────────────────────────────────
print("Sending to LLM...")
response = client.chat.completions.create(
    model=os.getenv("OLLAMA_MODEL"),
    messages=[
        {"role": "system", "content": compiled_system},
        {"role": "user",   "content": "What are the top 3 test types for an API?"},
    ],
    temperature=temperature,
)
print("\nResponse:")
print(response.choices[0].message.content)
BASH run it
$ python 02_use_prompts.py
config is your model settings store: Notice that system_prompt_obj.config returns the dict you passed at creation — {"model": "llama3.2", "temperature": 0.5}. This means your model parameters live alongside the prompt in Langfuse, not in code. When someone updates the temperature in the UI, your app picks it up automatically at next fetch — another zero-deploy change.

Link prompts to traces
03_linked_traces.py — see which version caused which output

This is where prompt management becomes truly powerful. By linking a prompt version to a generation, Langfuse shows you — per trace — exactly which prompt version was used. When you later look at a bad output, you can see if it was caused by v1 or v2 of your prompt.

Create day2/03_linked_traces.py:

PYTHON day2/03_linked_traces.py 
from dotenv import load_dotenv
import os
load_dotenv("../.env")

from langfuse import get_client, observe, propagate_attributes
from openai import OpenAI

langfuse = get_client()
client   = OpenAI(base_url=os.getenv("OLLAMA_BASE_URL"), api_key="ollama")


@observe(name="qa-answer", as_type="generation",
         capture_input=False, capture_output=False)
def answer_with_prompt(question: str) -> str:

    # 1. Fetch the production prompt from Langfuse
    prompt_obj = langfuse.get_prompt("qa-architect-system")   # production label

    # 2. Compile with variable values
    system = prompt_obj.compile(
        experience_years="20",
        specialisation="test case design and quality strategy",
        style="concise, structured bullet points",
    )

    # 3. Update generation with model + input + PROMPT LINK
    #    langfuse_prompt links this generation to the exact prompt version
    langfuse.update_current_generation(
        model=os.getenv("OLLAMA_MODEL"),
        model_parameters={
            "temperature": prompt_obj.config.get("temperature", 0.5),
        },
        input=[
            {"role": "system", "content": system},
            {"role": "user",   "content": question},
        ],
        # ✅ KEY: this links the generation to the prompt version
        prompt=prompt_obj,
    )

    # 4. Call the LLM
    response = client.chat.completions.create(
        model=os.getenv("OLLAMA_MODEL"),
        messages=[
            {"role": "system", "content": system},
            {"role": "user",   "content": question},
        ],
        temperature=prompt_obj.config.get("temperature", 0.5),
    )
    reply = response.choices[0].message.content

    # 5. Log output and token usage
    langfuse.update_current_generation(
        output=reply,
        usage={
            "input":  response.usage.prompt_tokens,
            "output": response.usage.completion_tokens,
        },
    )
    return reply


if __name__ == "__main__":
    questions = [
        "What is the difference between verification and validation?",
        "How do you decide test coverage thresholds for a new feature?",
        "What makes a good bug report?",
    ]

    for q in questions:
        with propagate_attributes(
            trace_name="Prompt-linked QA answer",
            user_id="user-123",
            tags=["prompt-linked", "day2"],
            metadata={"tutorial": "day2-hour1"},
        ):
            answer = answer_with_prompt(q)
            print(f"Q: {q}")
            print(f"A: {answer[:150]}...\n")

    langfuse.flush()
    print("✓ Check traces in Langfuse — each generation shows 'Prompt: qa-architect-system v1'")
BASH run it
$ python 03_linked_traces.py
What to look for in the dashboard: Click a trace → click the qa-answer generation. In the generation panel you'll see a new Prompt field showing qa-architect-system · v1 as a clickable link. From there you can jump directly to the prompt in Prompt Management. This is the version-to-output link that makes debugging trivial.

Version & deploy without code changes
04_deploy_workflow.py — two versions, zero redeploys

This is the core value demo. We'll create v2 of the prompt via Python, promote it to production, and show that the running app picks it up automatically.

PYTHON day2/04_deploy_workflow.py 
from dotenv import load_dotenv
import os, time
load_dotenv("../.env")

from langfuse import get_client
from openai import OpenAI

langfuse = get_client()
client   = OpenAI(base_url=os.getenv("OLLAMA_BASE_URL"), api_key="ollama")

QUESTION = "What is exploratory testing and when should I use it?"


def run_with_current_production_prompt(label: str = "production") -> dict:
    """Fetch production prompt, compile, call LLM. Returns version used + answer."""
    prompt_obj = langfuse.get_prompt("qa-architect-system", label=label,
                                       cache_ttl_seconds=0)  # bypass cache for demo
    system = prompt_obj.compile(
        experience_years="20",
        specialisation="exploratory and session-based testing",
        style="practical",
    )
    response = client.chat.completions.create(
        model=os.getenv("OLLAMA_MODEL"),
        messages=[
            {"role": "system", "content": system},
            {"role": "user",   "content": QUESTION},
        ],
    )
    return {
        "version": prompt_obj.version,
        "answer":  response.choices[0].message.content[:200],
    }


# ── PHASE 1: Run with current production (v1) ────────────────────
print("=== PHASE 1: Running with current production prompt ===")
result_v1 = run_with_current_production_prompt()
print(f"Prompt version used : v{result_v1['version']}")
print(f"Answer snippet      : {result_v1['answer']}...\n")


# ── PHASE 2: Create v2 with improved prompt ──────────────────────
print("=== PHASE 2: Creating improved v2 prompt ===")
v2 = langfuse.create_prompt(
    name="qa-architect-system",   # same name = new version of existing prompt
    type="text",
    prompt=(
        "You are a senior QA Architect with {{experience_years}} years of "
        "deep expertise in {{specialisation}}. "
        "IMPROVED v2: Always structure your answer with: "
        "1) Definition 2) When to use 3) Concrete example 4) Common mistakes. "
        "Communication style: {{style}}."
    ),
    config={"model": "llama3.2", "temperature": 0.4},
    labels=["staging"],   # push to staging first, NOT production yet
)
print(f"Created: {v2.name} v{v2.version} (labels: {v2.labels})\n")


# ── PHASE 3: Promote v2 to production ────────────────────────────
print("=== PHASE 3: Promoting v2 to production ===")
# In a real workflow you'd test v2 in staging first.
# Here we promote immediately for the demo.
langfuse.create_prompt(
    name="qa-architect-system",
    type="text",
    prompt=v2.prompt,
    config=v2.config,
    labels=["production"],   # now this version is production
)
print("Production label moved to v2.\n")


# ── PHASE 4: Same code, new prompt, no redeploy ──────────────────
print("=== PHASE 4: Same code — picks up new production prompt ===")
result_v2 = run_with_current_production_prompt()
print(f"Prompt version used : v{result_v2['version']}")
print(f"Answer snippet      : {result_v2['answer']}...")

print("\n✓ Same function, different prompt version — zero code change!")
langfuse.flush()
BASH run it
$ python 04_deploy_workflow.py
What you'll see: Phase 1 uses v1. Phase 4 uses v2 (or v3 if you ran create_prompts.py twice). The answer in Phase 4 should be structured differently because the new prompt instructs it to use numbered sections. Same code. Different output. Different prompt version. This is the entire value proposition.

Fallback & caching
production safety when Langfuse is unreachable

Langfuse adds zero latency to your app because prompts are cached locally. But what happens if Langfuse Cloud is temporarily unavailable when your service restarts (cache is cold)?

The caching system: First call fetches from Langfuse and caches for 60 seconds (default TTL). Subsequent calls within that window return from local cache instantly — zero network round-trip. Configure TTL: get_prompt("name", cache_ttl_seconds=300) for 5 minutes. Set to 0 to bypass (good for dev/testing). Set to a large number (e.g. 86400) to cache for a day.

Use fallback to keep your app running if Langfuse is unreachable:

PYTHON fallback pattern — add this to any production function 
# SAFE PRODUCTION PATTERN: always have a fallback prompt string
FALLBACK_SYSTEM = (
    "You are an experienced QA Architect. "
    "Provide practical, actionable advice."
)

def get_system_prompt_safely(**variables) -> tuple[str, int | None]:
    """Returns (compiled_prompt, prompt_version). Falls back gracefully."""
    try:
        prompt_obj = langfuse.get_prompt(
            "qa-architect-system",
            cache_ttl_seconds=300,   # cache 5 min in production
            fallback=FALLBACK_SYSTEM,  # used if Langfuse is unreachable
        )
        compiled = prompt_obj.compile(**variables)
        return compiled, prompt_obj.version

    except Exception as e:
        # Langfuse down AND cache cold — use hardcoded fallback
        print(f"[WARN] Langfuse unavailable: {e}. Using fallback prompt.")
        return FALLBACK_SYSTEM, None


# Usage:
# system, version = get_system_prompt_safely(
#     experience_years="20",
#     specialisation="test automation",
#     style="concise"
# )
Don't pass cache_ttl_seconds=0 in production. It makes every request hit Langfuse Cloud, adding 50-200ms of latency to every LLM call. Use it only in dev/testing when you want to see prompt changes immediately without waiting for the cache to expire.

Hour 1 mini-project: prompt-managed chatbot
05_prompt_chatbot.py — Day 1 capstone + prompt management

Upgrade the Day 1 capstone chatbot so every LLM call pulls its system prompt from Langfuse instead of a hardcoded string. The QA team can now iterate on the chatbot's persona without touching Python code.

PYTHON day2/05_prompt_chatbot.py 
from dotenv import load_dotenv
import os, uuid
load_dotenv("../.env")

from langfuse import observe, get_client, propagate_attributes
from openai import OpenAI

langfuse = get_client()
client   = OpenAI(base_url=os.getenv("OLLAMA_BASE_URL"), api_key="ollama")

FALLBACK_SYSTEM = "You are an experienced QA Architect. Provide practical advice."


@observe(name="prompt-fetch")
def fetch_system_prompt(specialisation: str, style: str) -> tuple:
    """Fetch & compile system prompt from Langfuse. Returns (text, prompt_obj)."""
    try:
        obj = langfuse.get_prompt(
            "qa-architect-system",
            cache_ttl_seconds=300,
            fallback=FALLBACK_SYSTEM,
        )
        text = obj.compile(
            experience_years="20",
            specialisation=specialisation,
            style=style,
        )
        langfuse.update_current_span(
            output={"prompt_name": obj.name, "version": str(obj.version)},
        )
        return text, obj
    except Exception:
        return FALLBACK_SYSTEM, None


@observe(name="chat-llm-call", as_type="generation",
         capture_input=False, capture_output=False)
def call_llm(messages: list, prompt_obj, turn: int) -> str:
    langfuse.update_current_generation(
        model=os.getenv("OLLAMA_MODEL"),
        model_parameters={"temperature": 0.6},
        input=messages,
        prompt=prompt_obj,           # ← links generation to prompt version
        metadata={"turn": str(turn)},
    )
    response = client.chat.completions.create(
        model=os.getenv("OLLAMA_MODEL"),
        messages=messages,
        temperature=0.6,
    )
    reply = response.choices[0].message.content
    langfuse.update_current_generation(
        output=reply,
        usage={
            "input":  response.usage.prompt_tokens,
            "output": response.usage.completion_tokens,
        },
    )
    return reply


@observe(name="chat-turn")
def process_turn(messages: list, system_text: str,
                  prompt_obj, turn: int) -> str:
    return call_llm(messages, prompt_obj, turn)


class PromptManagedChatbot:
    def __init__(self, user_id: str = "user-123"):
        self.user_id    = user_id
        self.session_id = f"pm-{uuid.uuid4().hex[:8]}"
        self.turn       = 0

        # Fetch prompt once at startup — refreshes from cache every 5 min
        self.system_text, self.prompt_obj = fetch_system_prompt(
            specialisation="test strategy and quality engineering",
            style="concise, structured, with examples",
        )
        self.history = [{"role": "system", "content": self.system_text}]

        v = getattr(self.prompt_obj, "version", "fallback")
        print(f"\n{'='*52}")
        print(f"  QA Architect · Prompt-Managed Chatbot")
        print(f"  Prompt  : qa-architect-system v{v}")
        print(f"  Session : {self.session_id}")
        print(f"  Type 'quit' to exit")
        print(f"{'='*52}\n")

    def chat(self, user_input: str) -> str:
        self.turn += 1
        self.history.append({"role": "user", "content": user_input})

        with propagate_attributes(
            trace_name=f"PM-Turn {self.turn}",
            session_id=self.session_id,
            user_id=self.user_id,
            tags=["prompt-managed", "day2"],
            metadata={"turn": str(self.turn), "tutorial": "day2-hour1"},
        ):
            reply = process_turn(
                self.history, self.system_text,
                self.prompt_obj, self.turn
            )

        self.history.append({"role": "assistant", "content": reply})
        return reply

    def end(self):
        langfuse.flush()
        print(f"\n✓ {self.turn} turns traced → Sessions → {self.session_id}")


if __name__ == "__main__":
    bot = PromptManagedChatbot(user_id="user-123")
    while True:
        q = input("You: ").strip()
        if q.lower() in ("quit", "q"):
            break
        if q:
            print(f"\nQA Architect: {bot.chat(q)}\n")
    bot.end()
